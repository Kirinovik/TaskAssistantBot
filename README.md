# Task Assistant Bot
Ассистент в Telegram для быстрой постановки задач сотрудникам. Бот распознает голос, извлекает суть задачи с помощью ИИ и отправляет официальное уведомление на электронную почту исполнителя.

## Основные возможности
* Авторизация: Доступ к функциям бота защищен паролем.
* Голосовое управление: Поддержка Whisper API (через Groq) для преобразования голосовых сообщений в текст.
* ИИ-парсинг задач: Использование LLM (Llama 3.3) для автоматического определения исполнителя, описания задачи и крайнего срока (Deadline) из произвольного текста.
* Email-интеграция: Автоматическая отправка писем через SMTP (Яндекс) с использованием базы контактов.
* Управление историей: Просмотр списка активных задач, ручное напоминание и отмена задачи с уведомлением исполнителя по почте.

## Технологический стек
* Язык: C# (.NET 8.0)
* Библиотека Telegram: Telegram.Bot
* База данных: SQLite (Microsoft.Data.Sqlite)
* Почта: MailKit (SMTP SSL)
* ASR (Речь в текст): Groq API (Whisper-large-v3)
* LLM (Анализ текста): Groq API (Llama-3.3-70b-versatile)

## Установка и запуск

### Способ 1: Использование Visual Studio 2022
1.  **Клонирование проекта:**
    * Запустите Visual Studio 2022.
    * Выберите **Clone a repository** (Клонировать репозиторий) и вставьте URL этого проекта.
2.  **Настройка конфигурации (User Secrets):**
    * Чтобы не хранить пароли в открытых файлах проекта, нажмите **правой кнопкой мыши по проекту** в Обозревателе решений.
    * Выберите пункт **Manage User Secrets** (Управление секретами пользователя).
    * В открывшийся файл `secrets.json` вставьте блок настроек (см. раздел "Описание секретов" ниже).
3.  **Запуск:**
    * Нажмите кнопку **Start** (зеленая стрелка) или клавишу `F5`.
    * Консоль отобразит процесс инициализации базы данных и статус подключения к Telegram.

### Способ 2: Использование командной строки (CLI)

1.  Откройте терминал в папке с проектом.
2.  Создайте файл `appsettings.json` на основе примера.
3.  Выполните команды:
    ```bash
    dotnet restore
    dotnet build -c Release
    dotnet run --project TaskAssistantBot.csproj
    ```

## Команды бота
* /start — Проверка статуса авторизации и приветствие.
* /help — Полный список доступных команд.
* /list — Показать последние 25 активных задач с их ID.
* /add [Имя] [Email] — Добавление нового сотрудника в базу.
* /contacts — Список всех сохраненных контактов.
* /cancel [ID] — Отмена задачи (удаление из списка + письмо сотруднику).
* /remind [Имя] — Повторная отправка письма по последней задаче сотрудника.
* /logout — Выход из сессии.

## Описание переменных (Secrets)

Все конфигурационные данные и ключи доступа хранятся в формате JSON. Бот автоматически считывает их из файла `secret.json`.

### Таблица параметров конфигурации

| Секция | Переменная | Описание |
| :--- | :--- | :--- |
| **BotSettings** | `BotToken` | Уникальный токен вашего бота, полученный от [@BotFather](https://t.me/BotFather). |
| | `AdminPassword` | Пароль для авторизации в боте. При неверном вводе бот выдаст ошибку. |
| **Groq** | `ApiKey` | Ключ API для доступа к моделям Whisper (речь) и Llama 3.3 (текст) через [Groq Cloud](https://console.groq.com/). |
| **EmailSettings**| `SmtpServer` | Адрес SMTP-сервера вашей почты (например, `smtp.yandex.ru`, `smtp.gmail.com`). |
| | `Port` | Порт для подключения. Рекомендуется `465` (для SSL) или `587` (для TLS). |
| | `BotEmail` | Электронная почта, с которой будут уходить уведомления исполнителям. |
| | `BotPassword` | Специальный **пароль приложения**, сгенерированный в настройках безопасности вашей почты. |
